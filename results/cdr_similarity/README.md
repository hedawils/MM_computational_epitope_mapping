# CDR-only similarity analysis (BLOSUM62)

Generated by [../../scripts/cdr_similarity_analysis.py](../../scripts/cdr_similarity_analysis.py).
Restricts the comparison in
[../sequence_similarity](../sequence_similarity/README.md) to CDR loops only, and
scores alignments with BLOSUM62 instead of plain match/mismatch identity.

## CDR boundary detection

There is no author-confirmed Kabat/IMGT numbering for these clones, so CDR1/2/3
boundaries are detected from conserved framework anchor motifs (regex), **verified
by hand against all 15 VH and 15 VL sequences** in
[../../data/processed/variable_domains.fasta](../../data/processed/variable_domains.fasta).
This is a heuristic tuned to this specific dataset, not a general-purpose
Kabat/Chothia/IMGT numbering tool — treat CDR boundaries as approximate.

- The first and second cysteine in each domain reliably bound FR1/CDR1 and
  FR3/CDR3 (standard Ig fold; exactly 2 Cys per domain in all 15 clones).
- VH: FR2 starts at `W[VF]RQAPG`, CDR-H2 starts after `LE[WY]IG`/`LQWIG`, FR3
  starts at `R[FSA]T[ILV][ST]`, FR4 starts at `WG.GT`.
- VL (kappa + lambda): FR2 starts at `W[FYI]QQ`, CDR-L2 starts after
  `LLIY`/`LLIH`/`LVIY`/`LLIF`/`LMQL`, FR3 starts at `G.P[SD]R`, FR4 starts at
  `FG.GT`.
- Every extracted CDR was sanity-checked to fall in a plausible length range
  (2-30 aa); the script raises an error rather than silently emitting a bad
  boundary if a clone doesn't match.

Extracted CDR1/2/3 loops and per-chain CDR1+CDR2+CDR3 concatenations:
[../../data/processed/cdr_regions.fasta](../../data/processed/cdr_regions.fasta).

## Alignment

Concatenated CDRs (VH: CDR-H1+H2+H3; VL: CDR-L1+L2+L3) are aligned pairwise with
`Bio.Align.PairwiseAligner` (global alignment, BLOSUM62 substitution matrix,
affine gap penalties: open -10, extend -0.5). Percent identity = identical
aligned positions / total alignment length (gaps counted as mismatches).

## Outputs

- `cdr_identity_matrix_VH.csv`, `cdr_identity_matrix_VL.csv` — 15x15 %identity
- `cdr_dendrogram_VH.png`, `cdr_dendrogram_VL.png` — average-linkage clustering
- `cdr_similarity_summary.txt` — same-target vs. different-target mean identity

## Result vs. full-domain comparison

Restricting to CDRs and BLOSUM62 scoring widens the same-target vs.
different-target gap seen in the [full-domain analysis](../sequence_similarity/README.md)
(VH 71.0% vs 70.0%, VL 76.0% vs 75.0%) to:

- VH: 54.3% (same-target) vs. 49.7% (different-target)
- VL: 61.6% (same-target) vs. 56.8% (different-target)

Same-target clones are somewhat more CDR-similar than different-target clones,
as expected once framework identity is removed from the comparison — but the
overlap is still large, so CDR similarity alone would not reliably predict
target antigen for this panel.
